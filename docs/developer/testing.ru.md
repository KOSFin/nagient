# Разработчик: тестирование плагинов

Язык: [English](testing.md) | Русский

Каждый репозиторий плагина должен запускать тесты до публикации тега.

## Python-плагин

Используйте отдельное окружение репозитория и установите ядро как тестовую
зависимость:

```bash
python -m venv .venv
. .venv/bin/activate
python -m pip install -e .
python -m unittest discover -s tests
```

Минимальный набор проверяет манифест, неверную конфигурацию, secret refs,
нормализацию одного входящего события и исходящий запрос через mock. Не вызывайте
Telegram, GitHub или LLM из unit-тестов.

## Process-плагин

Проверяйте JSON-протокол fixture-запросами:

```bash
printf '%s\n' '{"protocol":"nagient.process.v1","method":"healthcheck"}' | ./plugin
```

Процесс должен вернуть один JSON-объект и не писать логи в stdout; диагностика
выводится в stderr.

## Чеклист релиза

1. Закрепите зависимости и запустите тесты на Python 3.11 и 3.12.
2. Установите репозиторий в чистый runtime командой `nagient plugin install <repo>#<tag>`.
3. Запустите `nagient preflight --format json` и проверьте ошибки.
4. Добавляйте репозиторий и тег в проверенный каталог только после ревью.
